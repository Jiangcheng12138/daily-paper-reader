## 摘要
公共神经生理数据集日益可访问，但仍难以重用：将一个数据集转化为训练好的模型仍然需要数千行代码用于下载、加载、格式修复、窗口划分和评估，而一个符合元数据标准的数据集仍可能加载失败。EEG-Dash是一个软件资源，它编录了来自OpenNeuro和NEMAR档案的791个公共存档记录（39,778名参与者，超过86,051小时），涵盖脑电图（EEG）、脑磁图（MEG）、颅内脑电图（iEEG）、肌电图（EMG）和功能性近红外光谱（fNIRS）。它将每个数据集暴露为一个可导入、可查询的类，保留信号属性，无需自定义代码即可加载到机器学习工作流程中，将信号处理委托给MNE-Python，窗口划分委托给Braindecode，格式合规性委托给官方脑成像数据结构（BIDS）验证器。一个元数据优先的注册表增加了语义搜索、格式修复层、从每个源出版物中提取的自动数据集级标签，以及一个特征提取框架。该目录包含每条记录的加载能力和合规性元数据，支持基准测试、模型开发和跨数据集分析。

## Abstract
Public neurophysiological datasets are increasingly accessible but remain hard to reuse: turning one into a trained model still takes thousands of lines of code for download, loading, format repair, windowing, and evaluation, and a dataset that meets metadata standards can still fail to load. EEG-Dash is a software resource that catalogues 791 publicly archived recordings (39,778 participants, over 86,051 hours) spanning electroencephalography (EEG), magnetoencephalography (MEG), intracranial EEG (iEEG), electromyography (EMG), and functional near-infrared spectroscopy (fNIRS) from the OpenNeuro and NEMAR archives. It exposes each dataset as an importable, queryable class that preserves signal attributes and loads into machine-learning workflows without custom code, delegating signal handling to MNE-Python, windowing to Braindecode, and format compliance to the official Brain Imaging Data Structure (BIDS) validator. A metadata-first registry adds semantic search, a format-repair layer, automatic dataset-level tags drawn from each source publication, and a feature-extraction framework. The catalogue, with per-record loadability and compliance metadata, supports benchmarking, model development, and cross-dataset analysis.

### 1. 核心问题与整体含义

- **问题**：公共神经生理数据集（EEG、MEG、iEEG、fNIRS、EMG）虽然开放可访问（符合FAIR原则中的可发现和可访问），但复用至机器学习流程时依旧困难。研究者通常需要数千行自定义代码处理下载、格式修复、窗口划分、标签对齐和评估，且即使符合BIDS元数据标准的数据集也可能由于加载器与标准的漂移而加载失败。
- **整体含义**：这种“可用性鸿沟”严重阻碍了跨数据集的基础模型训练、基准测试和可重复研究。作者提出EEG-Dash平台，通过元数据优先的注册表、自动格式修复层、以及委托现有成熟工具（MNE-Python、Braindecode、官方BIDS验证器），将大规模公开神经生理数据转化为可直接用于PyTorch机器学习流程的数据集类，从而显著降低复用门槛。

### 2. 方法论

- **核心思想**：**元数据优先的懒加载**。用户仅下载模型实际消耗的信号，而非整个档案（数百TB）。元数据先于信号被索引，支持离线过滤和查询。
- **关键技术细节**：
    - **委托架构**：信号处理委托给MNE-Python；窗口化和深度学习委托给Braindecode/PyTorch；BIDS解析委托给MNE-BIDS；BIDS合规性检查委托给官方BIDS验证器；HPC调度委托给NSG。EEG-Dash自身只拥有注册表层、格式修复层和特征提取框架。
    - **元数据模型**：两级文档模型——`Dataset Document`（研究级元数据：人口统计、模态、认知域等）和`Record Document`（文件级元数据：BIDS实体、通道数、采样率、时长等）。支持通过REST API进行语义搜索和记录级过滤。
    - **修复层**：四个缺陷类别——文件指针缺陷（BrainVision头文件路径错误）、元数据缺陷（JSON/TSV值类型错误）、结构缺陷（缺失事件/通道表）、格式特定缺陷（EDF/MEF3/FIF读取失败）。每个缺陷通过检测-修复管道进行最小校正，不改变原始信号。
    - **特征提取模块**（`eegdash.features`）：基于共享计算树的特征提取，提供46个特征（34个单变量、11个双变量、1个训练型CSP）和10个预处理器，支持扩展。
    - **ML就绪访问**：每个记录转换为内存映射的Braindecode格式（继承PyTorch Dataset），实现懒加载和缓存。

### 6. 主要结论与发现

- **BIDS合规性较低**：仅33.2%的OpenNeuro EEG数据集通过官方BIDS验证器（182/548）。
- **加载成功率较好**：标准BIDS加载器可直接加载95.0%的记录（108,322/114,031），修复层额外恢复0.8%达到95.8%。
- **合规性与加载能力几乎不相关**：斯皮尔曼ρ = -0.05（95%CI [-0.14, 0.04]，p≈0.26，n=503）。有效验证器-加载器分离，不能以其中一个预测另一个。
- **跨表揭示**：219个验证失败的数据集仍可完全加载；37个验证通过的数据集加载失败或部分失败（其中12个完全不可加载）。
- **端到端ML就绪**：所有791个数据集可通过三行代码（导入类、包装为DataLoader、迭代）直接用于PyTorch训练，无需自定义数据加载代码。

### 7. 优点

- **元数据优先设计**：允许用户在下载原始信号前进行过滤和选择，大幅节省带宽和存储。
- **委托-贡献模式**：EEG-Dash不重新实现基础功能，而是将修复贡献回上游（MNE-BIDS等），使整个生态受益，避免闭源分叉。
- **开放、可审计**：发布每条记录的加载和合规性结果，支持可重复的格式监控。
- **丰富的界面**：提供Web浏览器（eegdash.org）和电极图查看器，降低探索门槛。
- **特征提取共享计算**：特征树避免重复计算，支持scikit-learn接口和自定义特征。
- **自动标签和语义搜索**：利用LLM辅助自动生成人口统计、模态、认知域标签，提高发现效率。
- **持续集成保证**：每个版本检查所有数据集类的接口一致性，确保三条代码即可使用。

### 8. 不足与局限

- **事件注释覆盖不全**：v0.8.2中HED（层级事件描述符）标签不完整，跨研究的范式查询受限。
- **信号质量未标准化**：修复层仅处理格式缺陷，不校正采集伪迹（电极噪声、线路干扰），用户需自行清洗。
- **模态不平衡**：EEG数据集占主导（659/791），iEEG、MEG、fNIRS、EMG为少数；健康参与者远多于临床群体，跨模态和临床结论需谨慎。
- **审计范围有限**：仅审计OpenNeuro EEG，未审计其他模态或NEMAR等其他来源。NEMAR数据经过平台治理，理论上合规性较高，但未在论文中量化。
- **感知属性未测量**：接口的“可读性”和“可理解性”需要用户Likert评分，尚未进行。
- **相关性而非因果性**：合规-加载关系的关联分析不能确证原因，可能受其他混淆因素（如数据集大小、复杂程度）影响。
- **非临床用途**：平台仅为研究基准测试和机器学习评估提供基础设施，不适用于临床诊断。
- **版本依赖性**：加载器和验证器均持续演化，数据集的状态会随工具链版本漂移，需要定期重新审计。
